1. entity - relationships
1-1. 데이터베이스에서 관리하고자 하는 객체를 의미합니다.
1-2. 엔티티는 테이블을 의미합니다.
1-3. 속성 : 엔티티의 세부 정보, 테이블의 칼럼에 해당합니다.
1-4. 엔티티 내에는 기본키가 존재합니다.
1-5. 관계 : 다른 엔티티와 연관성을 나타냅니다.


2. Pluggable DB
2-1. 하나의 컨테이너 DB안에 독립적인 데이터베이스 여러 개를 포함시킬 수 있는 구조를 의미합니다.

2-2. 구성요소
1. Container Database(CDB)
=> PDB를 포함하는 상위 데이터베이스입니다.
=> 데이터베이스의 공통 구조와 리소스를 관리합니다.
=> root container(CDB$ROOT)와 seed container(PDB$SEED)를 기본적으로 가집니다.

2. Pluggable Database(PDB)
=> 사용자가 실제 데이터를 저장하고 운영하는 독립적인 데이터베이스입니다.
=> 하나의 CDB 안에 여러 개의 PDB를 포함할 수 있습니다.
=> 독립적인 사용자, 데이터 스키마, 객체를 가질 수 있습니다.

** 각 PDB는 독립적인 DB처럼 동작합니다. (고유한 스키마, 테이블, 데이터)
** 각 PDB 간 데이터는 격리되어 있어 보안과 안정성이 높습니다.
