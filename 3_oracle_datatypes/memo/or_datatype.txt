1. varchar2 (size)
=> 사용한 만큼 메모리를 차지하는 문자열 타입입니다.

2. char
=> 할당된 메모리를 전부 사용하는 문자열 타입입니다.

3. number
=> 부동 소수점 38자리까지 저장할 수 있는 숫자 타입입니다.

4. date
=> 날짜 데이터 타입입니다.

5. Long, RAW, LONG RAW
=> 이전 버전과 호환성을 위해 사용하는 큰 메모리를 차지하는 문자 타입 and binary 타입입니다.
=> 추천하지 않습니다.

6. blob
=> 이미지, 비디오 사운드, 멀티미디어 파일 같은 비구조적 데이터를 저장하는 타입입니다. 128 TB

7. clob
=> 메모리를 많이 차지하는 문자열을 저장할 떄 사용하는 타입입니다.
=> long과 비슷하지만 더 용량이 크고 알고리즘이 더 좋습니다.

8. bfile
=> 데이터베이스 내부가 아니라 외부 파일 시스템에 저장된 파일을 참조하는 역할을 합니다.
=> 이미지, 비디오, PDF 파일 등을 데이터베이스에 저장하지 않고, 외부 디렉토리에 두고 
파일 경로만 데이터베이스에 저장하는 방식입니다.
=> 읽기 전용 데이터 타입 입니다.
=> bfile을 사용하려면 외부 파일이 있는 디렉토리 경로를 Oracle에 등록해야 합니다. 

8. rowid
=> base-64 숫자로, 테이블에서 한 행의 물리적 주소를 나타내는 타입입니다.
=> 16진법이고 행이 사라질 떄까지 존재합니다.
