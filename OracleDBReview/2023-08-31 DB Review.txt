속성 - 속성 - 칼럼

pk를 기본 식별자라고 한다. pk가 두 개인 테이블인 복합 식별자

도메인 => 제약조건/데이터 타입

관계 => 부모-자식 fk 자식의 pk가 부모의 fk라면 외래식별자라고 한다.

카디널리티 => 부모-자식간의 페어링 여부를 확인하여 일대일 대다대 일대다의 관계를 판단

옵셔널리티 => 테이블 두 개의 행들이 페어링 여부를 판단해서 전부면 필수관계 그렇지 않으면 선택 관계이다.
선택관계의 선에 가까운 곳에 동그라미 표시를 한다.

식별관계 => 부모의 pk가 자식의 fk로 내려와서 자식이 pk가 fk라면 식별관계 fk가 단순히 칼럼이라면 비식별관계

정규형 => 데이터 이상 현상을 제거하기 위한 지침
1 정규형 => 한 칼럼이 다중의 값을 가지고 있는 경우 (데이터 타입이 두 개인 경우를 떠올리자)
2 정규형 => pk에 따르지 않는 속성이 있는 경우 (pk로 상품을 찾을 수 없는 경우를 떠올리자)
3 정규형 => pk가 아닌데 pk처럼 행동하는 속성이 있고 거기에 따르는 속성이 있을 경우
